Module 01

Why complexity matters

An algorithmA precise, step-by-step procedure for solving a problem or completing a task. is just a recipe of steps. Two recipes can reach the same dish — but one might take 3 minutes and the other 3 hours. Complexity analysis is how we predict, before we even run the code, which one that will be.

Simulate a linear searchChecking items one at a time, in order, until the target is found. for one student's name in a list of different sizes. Press run and watch how the work grows.

Same task. Same computer. The only thing that changed is how many students there are — yet the work jumped from a handful of checks to a million. That relationship between input size and work is exactly what complexity analysis measures.

Module 10

Binary search — O(log n)

Binary search only works on a sorted array. Each step throws away half of the remaining possibilities — so the work needed grows only logarithmicallyEach additional element roughly doubles what a logarithmic algorithm can handle in the same number of steps — the opposite of exponential growth. with n.

Module 11

Linear search — O(n)

No shortcuts here — in the worst case, every single element must be checked once. Work grows in direct proportion to n.

Nested loops — O(n²)

A loop inside a loop means the inner loop runs completely, once for every iteration of the outer loop. Drag n and watch the operation count explode quadratically, not linearly.
